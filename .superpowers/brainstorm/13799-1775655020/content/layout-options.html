A) Stacked Sections

Everything scrolls vertically in priority order. Ticket info on top, project context below, actions at bottom. Simple, familiar, works like the rest of the app. You scroll down to see more context.

B) Ticket Hero + Tabbed Context

Ticket info is always pinned at top (sticky). Below it, horizontal tabs let you switch between Activity, Other Tickets, Work Orders, and Actions. Keeps ticket visible while you explore context. Less scrolling, more tapping.

C) Action-First + Expandable Context

Compact ticket header, then action buttons front and center. Project context lives in collapsible sections below — expand what you need, collapse what you don't. Optimized for fast resolution: see ticket, act, move on. Context is one tap away but doesn't clutter the view.
