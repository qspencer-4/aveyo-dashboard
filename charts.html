Chart 1 · Chart.js horizontal

Action Queue — Horizontal Bars

Every widget from your current home page, rendered as a horizontal bar chart sorted by count. Color-coded by your existing warn/critical thresholds. Tap a bar to drill into that list in Podio. Replaces the whole widget grid with one clean visual.

Chart 2 · Pure CSS

Pipeline Funnel — Relative Bars

Zero-dependency version of the same idea, using your existing BF2015 styling. Lightweight, no canvas, animates smoothly, works offline in the PWA. Each bar's length is relative to the largest stage.

Chart 3 · Chart.js stacked

Pipeline by Stage — Stacked

One bar per Podio app, broken down by sub-stage. Quick read on where the work lives inside each app. Same data you already have, segmented by the `milestone` field.

Chart 4 · Chart.js trend

Reviews Needed — 14 Day Trend

Bars showing daily "review needed" count over the last 14 days. Requires a small daily snapshot job on the API (Podio doesn't keep history). Trend bars let you see "are we keeping up or falling behind" at a glance — the thing widgets can't tell you.

Chart 5 · Chart.js grouped

SOW Funnel — Grouped

Grouped bars showing SOW app sub-views side by side: this week vs last week. Great for weekly reviews. Also requires the daily snapshot job.

Chart 6 · Pure CSS

Activity Heatmap — 14 Days × Stage

Each cell = one day × one stage. Darker = more activity. Shows you when things happened, not just totals. Built with a CSS grid, no library needed. Feeds naturally from the realtime webhook stream.
